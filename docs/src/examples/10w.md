# 10w+条数据示例

这个示例直接使用 `generateStressTestData({ minNodes: 100_000 })` 生成大数据，展示：

- 估算节点数
- 实际解析节点数
- 当前可见行数
- 10w 数据下的树列表渲染

<demo vue="../demos/jsonup-100k.vue" />

## 这个页面重点看什么

- 估算节点数和实际节点数是否一致
- 展开和折叠时 `visibleNodes` 的变化
- 虚拟滚动下实际渲染的 DOM 行数

## 为什么可以撑住大数据？

`jsonup` 在解析时会把 JSON 转成扁平节点，并记录每个节点的 `descendants`。当节点折叠时，可见节点计算可以直接跳过整段子树，而不是递归检查每一个隐藏节点。

```ts
if (node.expandable && !node.expanded) {
  index += node.descendants;
}
```

文档 demo 还使用了虚拟滚动组件，只渲染视口附近的行。这样即使文档里有 10w+ 节点，页面上真实存在的 DOM 行数也保持在一个较小范围。

## 适用建议

- 大 JSON 默认只展开根节点，避免首屏展示过多内容。
- 对列表或对象批量展开时，优先限制展开数量。
- 搜索、过滤和路径定位可以基于 `nodes` 全量节点执行；树形展示优先使用 `visibleNodes`。
